## ğŸ¯ One-Command Setup

### Prerequisites
- **Java 21** ([Download here](https://adoptium.net/temurin/releases/))
- **4 GB RAM** minimum

### Run Yaci Store v0.1.x

Choose one of the available pre-built applications:

```bash
# Option 1: All Stores (recommended for full functionality)
curl -L https://github.com/bloxbean/yaci-store/releases/download/v0.1.4/yaci-store-all-0.1.4.jar -o yaci-store-all.jar
java -jar yaci-store-all.jar

# Option 2: UTxO Indexer (lightweight option)
curl -L https://github.com/bloxbean/yaci-store/releases/download/v0.1.4/yaci-store-utxo-indexer-0.1.4.jar -o yaci-store-utxo.jar
java -jar yaci-store-utxo.jar

# Option 3: Docker Compose (full stack)
curl -L https://github.com/bloxbean/yaci-store/releases/download/v0.1.4/yaci-store-docker-compose-0.1.4.zip -o yaci-store-docker.zip
unzip yaci-store-docker.zip && cd yaci-store-docker-compose
docker-compose up
```

**ğŸ‰ That's it!** Yaci Store will:
1. Start with embedded H2 database (or PostgreSQL if using Docker)
2. Launch REST API on `http://localhost:8080`
3. Open API docs at `http://localhost:8080/swagger-ui.html`
4. Begin syncing with Cardano blockchain data

### Available Applications

- **yaci-store-all-0.1.4.jar** (95.6 MB): Complete indexer with all stores enabled
- **yaci-store-utxo-indexer-0.1.4.jar** (156 MB): UTxO-focused lightweight indexer  
- **yaci-store-aggregation-app-0.1.4.jar** (154 MB): Aggregation and analytics focused
- **Docker Compose**: Full stack with PostgreSQL database

## ğŸ” What's Included

### Features
- **Embedded H2 database** for quick setup
- **Blockchain sync capability** from Cardano network
- **REST API endpoints** for data access
- **Swagger UI** for API documentation

### Available APIs
- **Blocks API**: `/api/v1/blocks`
- **Transactions API**: `/api/v1/transactions`
- **Addresses API**: `/api/v1/addresses`
- **UTxOs API**: `/api/v1/utxos`
- **Epochs API**: `/api/v1/epochs`

### Example Queries
```bash
# Get latest blocks
curl http://localhost:8080/api/v1/blocks?page=0&size=10

# Get transactions for a specific block
curl http://localhost:8080/api/v1/blocks/{block-hash}/transactions

# Check API health
curl http://localhost:8080/actuator/health
```
